D&D Companion

My friends and I like to play D&D on a weekly basis. I eventually got tired of dealing with constantly writing on and erasing the paper character sheet since it would eventually become nearly illegible due to the pencil stains, as well as almost erasing holes right through it. So I decided to digitize it.

It was built for fifth edition and includes a digital character sheet with all of the standard PHB options; directories for armor, weapons, equipment, trinkets, and spells; and a "games" section that allows multiple players part of the same campaign to chat.

The first iteration of the project - the one that we use right now - is built in React with a Node + Express backend, Postgres for its database, and is hosted on Heroku. This was my first major project and the code is as bad as you'd expect. It's important to note that this was written specifically for our internal, personal use, so you won't find it to be very welcoming or pretty if you do decide to check it out.

I've learned a lot since I first made this app, so I recently started working on a second version of the app using Vue instead of React, but still with the Node + Express backend, and still with a Postgres database hosted on Heroku.

You can find the app here:
https://dnd.parkersprouse.me/

And the sources here:
https://github.com/parkersprouse/dnd-companion
https://github.com/parkersprouse/dnd-companion-v2

Scribble

Scribble is a web app that I made because I wanted something that allowed me to take quick notes whenever I needed them, or just wanted to keep a long-running list to be able to reference. It's simple to use and allows for tagging notes to make finding notes fairly quick and easy.

After making it and using it for a while, I've determined that the design of the site is lacking in a lot of ways. Similar to the D&D Companion app, I'm planning on rewriting Scribble using better practices and giving it a more user-friendly design. I'm not planning on switching the tech used for it though because it was originally made with Vue and that's what I'd use for the rewrite regardless.

It's built with a Vue frontend and a Node + Express backend, Postgres for its database, and is hosted on Heroku. The rewrite will keep this tech stack.

You can find the app here:
http://scribble.parkersprouse.me/

And the sources here:
https://github.com/parkersprouse/scribble
https://github.com/parkersprouse/scribble-v2

Recipe Manager

My girlfriend and I like to do a lot of cooking, and that requires hunting down a variety of recipes that we inevitably have to keep track of. Instead of just bookmarking them and hoping they never change, move, or get deleted, I decided to make an app that lets us copy a recipe to our own personal store so that we always know where to find it (and let's us skip the life stories and ads that we don't care about). You're of course welcome to use it yourself, but I make no promises.

This is one of the first apps I made and doesn't follow very many good practices, but it works. Unlike the above apps, this one is not scheduled for a rewrite of any kind. It could definitely do with one, as well as some tweaks to the front-end design, but I actually find it fairly usable without very many issues.

It's built with a Vue frontend and a Node + Express backend, Postgres for its database, and is hosted on Heroku.

You can find the app here:
http://recipes.parkersprouse.me/

And the source here:
https://github.com/parkersprouse/recipe-manager

RSS Feed Reader

After having a variety of different news sites that I like to keep track of (politics, music, video games), I decided I wanted a way to keep track of all of them in one place instead of visiting them one by one and seeing what was new. So I took the opportunity and built my RSS Feed Reader, which lets you plug in any number of RSS, Atom, or RDF feeds and view them all in real time simultaneously. There are no accounts to log in to, just a locally tracked list of your provided feeds.

I had wanted to give Electron a try for a while and figured the idea for this app would be a good excuse to give it a try. It's built in Electron with the vue-electron library so that I could write the interface as a Vue app. There's no backend or server-side database. Everything is stored locally in a JSON file.

You can download the app here:
https://github.com/parkersprouse/rss-reader/releases/tag/v1.0.0

You can view the source here:
https://github.com/parkersprouse/rss-reader
